# VS Code / PBIP import guide

This repository now includes a source-controlled Power BI Project shortcut and semantic model scaffold:

- `ccmr-powerbi-dashboard.pbip` opens the project shortcut.
- `ccmr-powerbi-dashboard.Report/definition.pbir` points the report to the local semantic model by path.
- `ccmr-powerbi-dashboard.SemanticModel/definition.pbism` declares the semantic model item.
- `ccmr-powerbi-dashboard.SemanticModel/model.bim` contains the import tables, Power Query M partitions, relationships, calculated helper dimensions, an initial `FactCCMRStatus` calculated table, a `_Measures` table, and starter RLS roles.
- `.vscode/settings.json` maps Power BI/Fabric JSON schemas so VS Code online can validate PBIP, PBIR, PBISM, and BIM files.

## How to use

1. Open this repository in VS Code online or local VS Code.
2. Review `ccmr-powerbi-dashboard.SemanticModel/model.bim`; the semantic model is text-editable JSON/TMSL.
3. If using Power BI Desktop, open `ccmr-powerbi-dashboard.pbip` or `ccmr-powerbi-dashboard.Report/definition.pbir`.
4. In Power BI Desktop, set the `pDataFolder` parameter to the full local path of `sample_data/` with a trailing slash if the relative default does not resolve on your machine.
5. Refresh preview/apply changes. The model should create import tables from the sample CSV files, calculated dimensions, relationships, and starter measures.
6. Build report pages using `report_layout/` or save the project as PBIX if a single binary file is needed.

## Notes

Power BI Desktop project files are still a preview capability. Microsoft documentation notes that PBIP files store report and semantic model definitions as text folders and can be edited with VS Code, but some report metadata is best generated by Power BI Desktop. This scaffold focuses on the semantic model, data loading, relationships, and measures so a developer can assemble the report quickly.
